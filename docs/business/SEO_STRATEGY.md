## On-Page SEO Best Practices

### Meta Tags Template

```html
<!-- Primary Meta -->
<title>InterNull | [Page Title] | Institutional Privacy Middleware</title>
<meta name="description" content="[120-160 chars describing page content and benefit]">
<meta name="keywords" content="[5-7 primary keywords]">

<!-- Canonical -->
<link rel="canonical" href="https://internull.io/page-path">

<!-- Open Graph -->
<meta property="og:title" content="[Same as title or shorter]">
<meta property="og:description" content="[120-160 chars]">
<meta property="og:image" content="https://internull.io/og-[page].png">
<meta property="og:url" content="https://internull.io/page-path">
<meta property="og:type" content="website">

<!-- Twitter -->
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="[Title]">
<meta name="twitter:description" content="[Description]">
<meta name="twitter:image" content="https://internull.io/twitter-[page].png">
<meta name="twitter:creator" content="@InterNullIO">

<!-- Mobile -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta name="mobile-web-app-capable" content="yes">

<!-- Theme -->
<meta name="theme-color" content="#2563EB">
```

### Header Structure (H1-H3 Hierarchy)

```
H1: Main value proposition (1 per page)
    "Institutional Privacy Middleware for Crypto Trading"

  H2: Major sections
      "Why Institutions Need Privacy (But Not Anonymity)"
      "How InterNull Ensures Compliance"
      "Multi-Chain Privacy Architecture"

    H3: Subsections
        "Real-Time Audit Trail"
        "Per-Client Deployments"
        "OFAC Integration"
```

**Key Rules:**
- Exactly 1 H1 per page
- Use keywords naturally in H1/H2
- Maintain hierarchy (no H3 without H2)
- 60-70 words per section on average

## Technical SEO Checklist

### Page Speed
- [ ] LCP < 2.5s
- [ ] FID < 100ms
- [ ] CLS < 0.1
- [ ] Total page size < 150KB (JS)
- [ ] Images optimized (WebP, lazy-loaded)
- [ ] CSS minimized & critical path optimized
- [ ] No render-blocking resources

### Structured Data
- [ ] Organization schema (JSON-LD)
- [ ] Service/Product schema
- [ ] FAQ schema (on FAQ pages)
- [ ] BreadcrumbList schema
- [ ] Article/NewsArticle schema (blog posts)

### Site Architecture
- [ ] Sitemap.xml (auto-generated)
- [ ] robots.txt (allow all relevant paths)
- [ ] Mobile responsive (tested on iOS/Android)
- [ ] Proper 301 redirects (old â†’ new)
- [ ] No 404s in sitemap
- [ ] Breadcrumbs visible & functional

### Internal Linking
- [ ] Homepage links to top 5 resources
- [ ] Each page links to 3-5 relevant pages
- [ ] Anchor text uses keywords naturally
- [ ] No broken internal links
- [ ] Clear information architecture

### URL Structure
- [ ] Descriptive, keyword-relevant paths
- [ ] `/resources/use-case-crypto-funds` not `/blog/post-123`
- [ ] HTTPS (enforced)
- [ ] No unnecessary parameters
- [ ] Consistent (no trailing slashes vs trailing slashes)

### Accessibility (WCAG AA)
- [ ] All images have alt text
- [ ] Color contrast > 4.5:1
- [ ] Keyboard navigation works
- [ ] Focus indicators visible
- [ ] Screen reader tested

## Analytics & Monitoring

### Google Search Console Setup
```
1. Add property (internull.io)
2. Verify ownership (DNS record)
3. Submit sitemap.xml
4. Monitor impressions & CTR
5. Check coverage & indexing issues
6. Monitor Core Web Vitals
```

**Monthly Reviews:**
- Top performing pages (traffic, clicks, impressions)
- Average position by keyword cluster
- Click-through rate by keyword
- Impressions without clicks (optimize titles/descriptions)
- Indexing issues (if any)

```
Event tracking:
- Page views
```
